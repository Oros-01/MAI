# Описание задачи ЛР5

Вводится строка. Строка считается валидной, если заканчивается на подстроку "abc" (регистр не имеет значения).

Примеры:
- "xyzabc" — заканчивается (true)
- "abcdef" — не заканчивается (false)
- "abc" — заканчивается (true)
- "testABC" — заканчивается (true, т.к. регистр не важен)

# Структура данных

## Enum
```cpp
enum class State {Start, Found_C, Found_BC, Found_ABC, Invalid};
```

Состояния автомата:
- **Start** - начальное состояние
- **Found_C** - найден символ 'c' 
- **Found_BC** - найдены символы 'bc' 
- **Found_ABC** найдены символы 'abc' 
- **Invalid** - ошибочное состояние (строка не заканчивается на "abc")

## Struct
```cpp
struct Automaton {
    State status;
};
```

Структура для хранения состояния автомата.

# Функции

## isLetterC
```cpp
bool isLetterC(const char symbol)
```
Проверяет, является ли символ буквой 'c' (регистронезависимо). Возвращает true, если symbol равен 'c' или 'C'.

## isLetterB
```cpp
bool isLetterB(const char symbol)
```
Проверяет, является ли символ буквой 'b' (регистронезависимо). Возвращает true, если symbol равен 'b' или 'B'.

## isLetterA
```cpp
bool isLetterA(const char symbol)
```
Проверяет, является ли символ буквой 'b' (регистронезависимо). Возвращает true, если symbol равен 'a' или 'A'.


## transition
```cpp
void transition (Automaton& aut, const char input)
```
Изменяет состояние автомата в соответствии с введенным символом. Принимает ссылку на структуру Automaton и символ input, обновляет поле status согласно таблице переходов

## obrabotka
```cpp
bool obrabotka(const string &input)
```
Запускает проверку строки input с использованием функции transition. Возвращает:
- `true` - если строка заканчивается на подстроку "abc"
- `false` - если строка не заканчивается на "abc" или короче 3 символов

# Таблица переходов состояний

## Start (начальное состояние)
- Если символ 'c' (или 'C') → переход в состояние Found_C
- Если символ не 'c' → переход в состояние Invalid


## Found_C (найден символ 'c')
- Если следующий символ 'b' (или 'B') → переход в состояние Found_BC
- Если следующий символ не 'b' → переход в состояние Invalid

## Found_BC (найдены символы 'bc')
- Если следующий символ 'a' (или 'A') → переход в состояние Found_ABC
- Если следующий символ не 'a' → переход в состояние Invalid

## Found_ABC (успешное состояние)
- Нет переходов (автомат остается в состоянии успеха)

## Invalid (ошибочное состояние)
- Нет переходов (автомат остается в состоянии ошибки)

# Логика работы

1. Автомат начинает работу в состоянии Start
2. Проверка строки выполняется с конца (справа налево)
3. Если длина строки меньше 3 символов, функция obrabotka сразу возвращает false
4. Для каждого символа входной строки (начиная с последнего) вызывается функция transition
5. Если в процессе обработки автомат переходит в состояние Found_ABC или Invalid, проверка прекращается
6. После обработки проверяется финальное состояние:
    - Если финальное состояние Found_ABC → строка заканчивается на "abc" (true)
    - Если финальное состояние Invalid → строка не заканчивается на "abc" (false)

# Тесты

1. **INPUT = "xyzabc"** → результат `true`
2. **INPUT = "abcdef"** → результат `false`
3. **INPUT = "abc"** → результат `true`
4. **INPUT = "ab"** → результат `false`
5. **INPUT = "testABC"** → результат `true`
